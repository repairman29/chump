# INFRA-007: Ambient Stream Firing Audit

**Date:** 2026-04-20  
**Gap:** INFRA-007  
**Status:** FIXED — events now land in main repo's ambient.jsonl

## Finding

All five ambient-* scripts (`ambient-emit.sh`, `ambient-query.sh`, `ambient-rotate.sh`,
`ambient-watch.sh`, `broadcast.sh`) resolved the lock directory with:

```bash
REPO_ROOT="$(git rev-parse --show-toplevel 2>/dev/null || pwd)"
LOCK_DIR="$REPO_ROOT/.chump-locks"
```

In a linked worktree, `git rev-parse --show-toplevel` returns the **worktree path**
(e.g., `.claude/worktrees/eval-061`), not the main repo root. So every event written
from any worktree landed in `.claude/worktrees/<name>/.chump-locks/ambient.jsonl` —
an isolated file that no other agent reads.

The main repo's `.chump-locks/ambient.jsonl` received only:
- `session_start` and `bash_call` events emitted by `gap-claim.sh` and the Rust
  binary via hardcoded absolute paths
- Manual `cat >>` writes

It never received `commit`, `file_edit`, or `ALERT` events generated by git hooks
running in worktrees. 50+ PRs landed on 2026-04-20 with zero commit events in the
shared log.

## Verified Trace

Event path for a commit in a linked worktree:

```
git commit
  → .git/hooks/post-commit (symlink → scripts/git-hooks/post-commit)
    → $REPO_ROOT/scripts/ambient-emit.sh commit sha=… msg=…
      → LOCK_DIR = $REPO_ROOT/.chump-locks   ← BUG: worktree path, not main repo
      → writes to .claude/worktrees/<name>/.chump-locks/ambient.jsonl
      (main repo's ambient.jsonl untouched)
```

`git rev-parse --git-common-dir` returns:
- `.git` when run from the main repo
- `/abs/path/to/main/.git` when run from any linked worktree

This gives a reliable way to derive the main repo root.

## Fix

In all five scripts, added a `MAIN_REPO` derivation that uses `--git-common-dir`:

```bash
_GIT_COMMON="$(git rev-parse --git-common-dir 2>/dev/null || echo ".git")"
if [[ "$_GIT_COMMON" == ".git" ]]; then
    MAIN_REPO="$REPO_ROOT"
else
    MAIN_REPO="$(cd "$_GIT_COMMON/.." && pwd)"
fi
LOCK_DIR="$MAIN_REPO/.chump-locks"
```

`ambient-emit.sh` also preserves a separate `LOCAL_LOCK_DIR="$REPO_ROOT/.chump-locks"`
for `.wt-session-id` lookup so that each worktree retains a unique session ID (they
must not share the main repo's `.wt-session-id`).

## Verification

From `.claude/worktrees/infra-007`:

```

→ main .chump-locks/ambient.jsonl +1 line ✓

```

## Scope of Change

| File | Change |
|------|--------|
| `scripts/ambient-emit.sh` | MAIN_REPO + LOCAL_LOCK_DIR split |
| `scripts/ambient-query.sh` | MAIN_REPO for LOCK_DIR |
| `scripts/ambient-rotate.sh` | MAIN_REPO for LOCK_DIR |
| `scripts/ambient-watch.sh` | MAIN_REPO for LOCK_DIR |
| `scripts/broadcast.sh` | MAIN_REPO for LOCK_DIR |

The `post-commit` and other git hooks are unchanged — they call
`$REPO_ROOT/scripts/ambient-emit.sh` which is correct (the worktree's copy,
which now resolves LOCK_DIR correctly).

## What Was Not Broken

- `session_start` and `bash_call` events: emitted by `gap-claim.sh` / `broadcast.sh`
  called from the **main repo root** (where CLAUDE.md sessions run), so
  `--show-toplevel` returned the correct path — these always worked.
- The file-lock (flock) mechanism: not affected (macOS has no flock, uses plain `>>`).
- The `CHUMP_AMBIENT_LOG` override: still respected; overrides both paths.
