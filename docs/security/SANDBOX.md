# Agent-bash sandbox (INFRA-1454)

> **Status:** pilot — default off in v1, opt-in via `CHUMP_AGENT_SANDBOX=1`.
> Linux/podman/docker fallbacks land under follow-up gaps.

## Why this exists

Persona-5 (skeptic) interview surfaced the missing-sandbox surface as a hard
ship-blocker for any commercial Chump tier:

> Today agents run bash directly on the host filesystem inside a linked git
> worktree. Trust model collapses into "operator owns the machine" —
> defensible for solo dogfood, NOT defensible for any environment where the
> agent ingests third-party content or where the operator wouldn't run a
> `curl|bash` from a stranger. A single adversarial
> dependency-description-interpreted-as-instruction = CVE with the operator's
> name on it.

INFRA-1454 introduces the **syscall-restriction layer** that wraps agent
shell execution in an unprivileged sandbox so a hostile prompt can't escape
the worktree.

## What v1 ships

- `src/sandbox.rs` — `SandboxStatus`, `sandbox_runtime_status`,
  `agent_sandbox_enabled`, `build_profile`, `wrap_command`.
- `src/cli_tool.rs` — the agent-facing `run_cli` MCP tool routes its shell
  exec through `sandbox::wrap_command` (additive — no other call sites
  changed yet).
- `src/fleet_health.rs` / `chump health` — reports `sandbox_status` (active /
  available_not_enabled / disabled_by_operator / missing); flags **missing**
  as a 10-point health penalty so `chump fleet doctor` surfaces it.
- `scripts/ci/test-sandbox-isolation.sh` — source-contract assertions plus
  a real macOS integration: writes inside the worktree succeed, writes
  outside are blocked by `sandbox-exec`.

## How to turn it on

```bash
export CHUMP_AGENT_SANDBOX=1
chump fleet doctor          # confirms "sandbox active (runtime: macos/sandbox-exec)"
```

To force the legacy host-shell path back even when sandbox is otherwise
available (operator escape hatch):

```bash
export CHUMP_AGENT_UNSAFE_HOST_EXEC=1
# `chump health` reports: "sandbox runtime present but disabled by ..."
```

## What the v1 profile blocks (and what it doesn't)

The SBPL profile generated by `build_profile(worktree)` is:

```
(version 1)
(deny default)
(allow process-fork process-exec* signal mach-lookup ipc-posix-shm* sysctl-read iokit-open)
(allow file-read*)
(allow file-write*
  (subpath "<WORKTREE_ROOT>")
  (subpath "/private/tmp")
  (subpath "/private/var/folders")
  (subpath "/tmp"))
(allow network*)
```

**Blocks:** writes to the host filesystem outside the active worktree
(`~/Documents`, `/etc`, `~/.ssh`, `/usr/local/lib`, the user's other repos —
all denied).

**Does NOT block (yet — follow-up gaps):**
- **Network egress.** Permissive in v1 so the agent can call its LLM and
  `git fetch`. AC#3 (network allowlist) lands as a follow-up.
- **Reads from anywhere.** Permissive in v1 so the agent can read system
  libraries / git objects. Tightening reads is a separate exercise.
- **Container-level port-namespace isolation** for concurrent worktrees
  running docker-compose stacks. AC#8/#9 land under a follow-up.

## Detecting bypass

Every time the legacy host-shell path runs because the operator set
`CHUMP_AGENT_UNSAFE_HOST_EXEC=1`, `chump fleet doctor` surfaces it in the
"Sandbox:" line with the explicit reason. The audit trail is in `ambient.jsonl`
under `kind=fleet_health` (look for `sandbox_status=disabled_by_operator`).

## Roll-forward plan

1. **v1 (this PR)** — pilot lands default-off; operators opt in;
   metrics accumulate.
2. **v1.1** — flip default to on once `chump health` reports `active`
   for ≥7 consecutive days across the dev fleet with zero
   `kind=agent_unsafe_host_exec` events.
3. **v2** — tighten reads (file-read by-subpath); add a configurable
   network allowlist; ship Linux/podman fallback (follow-up gaps).
4. **v3** — container-level port-namespace isolation per worktree
   (AC#8/#9; depends on INFRA-1462 per-worktree init).

## Failure modes (v1)

- **`sandbox-exec` deprecated by Apple.** It is — and Apple still ships it
  on every macOS release through Sequoia. When it stops shipping we
  pivot to the `Endpoint Security` framework (much higher effort) or to
  Docker Desktop as the default macOS runtime.
- **Profile too tight, build breaks.** v1 errs permissive on reads/network
  on purpose. If a tool unexpectedly fails under sandbox, the operator can
  set `CHUMP_AGENT_UNSAFE_HOST_EXEC=1` to bypass while we tune.
- **Per-tool false denials.** Allowed-but-rare: cargo target dirs outside
  the worktree, parallel build caches. Track recurrences via
  `chump health --json` parses by sandbox-status.
