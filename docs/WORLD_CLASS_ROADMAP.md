# World-Class Dev Workflow Roadmap

**Status:** active (filed 2026-04-25, signed off by Jeff)
**Tracking gap:** INFRA-058
**Milestones:** INFRA-059 (M1) → INFRA-063 (M5)

## Why this exists

Chump's dispatcher and merge queue work, but the day-to-day cost is high:

- `docs/gaps.yaml` is a single hot file every PR appends to → tail-rebase
  conflicts cascade through the queue. Mass merges occasionally produce
  silent YAML corruption (observed 2026-04-25: three gap entries squashed
  into one `- id:` block during the #498/#494 merge sequence, blocking
  `cargo test --workspace` until detected).
- Agents pick a gap and dive into code without a planning step. Conflicts
  are discovered at PR time, not claim time.
- Related work fans out as independent parallel PRs that all touch the
  same files. There's no stacked-PR support.
- Cognitive-architecture changes live on long-lived branches. Bench data
  is collected against unmerged code, then we rebase, then we rebase again.
- `--skip-tests` is becoming routine in `bot-merge.sh` invocations because
  pre-existing main-state issues block honest test runs.

We've built a strong **reactive** layer (`queue-driver`, `resolve-gaps-conflict.py`,
checkpoint tags, ambient stream, lessons injection). The next layer is
**proactive** — prevent the conflict before it forms.

## Target shape

> A dispatched agent picks up a gap, gets a briefing, writes a 5-line plan
> with conflict prediction against open PRs, codes against a clean state,
> ships through merge queue, lands behind a flag for cognitive changes,
> and the user sees cycle-time + ship-rate on a dashboard. No appended
> hot files. No tail-rebase cascades. No `--skip-tests`. No 30+ stale
> worktrees. Continuous deploy to crates.io.

## Milestones

### M1 — SQLite-authoritative gap store (INFRA-059)

**Effort:** ~1 day · **Blocks:** M2, M3

Finish [INFRA-023](../scripts/) — `chump gap *` already exists but YAML
is still authoritative. Flip authority:

- All `chump gap` writes go to `.chump/state.db`.
- `docs/gaps.yaml` becomes a generated artifact. Regenerated on demand by
  `chump gap dump --out docs/gaps.yaml` for human diff review and committed
  alongside DB-mutating PRs.
- `chump gap reserve` + `chump gap claim` + `chump gap ship` are the only
  supported write paths.
- `gap-reserve.sh` / `gap-claim.sh` shell scripts become thin shims that
  call the binary (or are deprecated outright if no caller depends on the
  shell-only path).
- Pre-commit `duplicate-ID insert` guard becomes redundant (DB UNIQUE
  constraint enforces it) — keep the guard for the YAML diff but document
  it as belt-and-suspenders.

**Retirement (post-ship):** stop direct YAML edits in scripts and
agent prompts. Update `CLAUDE.md` + `AGENTS.md` to point at `chump gap`
exclusively.

**Acceptance:**
- `chump gap reserve` + `claim` + `ship` round-trip without touching YAML.
- `docs/gaps.yaml` is regenerated by `chump gap dump` and matches DB byte-for-byte.
- A simulated 5-way concurrent gap-reserve produces 5 distinct IDs (no collision).
- Pre-commit hook on `docs/gaps.yaml` no longer flags routine tail-appends.

### M2 — Plan-mode gate in the dispatcher (INFRA-060)

**Effort:** 2–3 days · **Depends on:** M1 (clean DB to query) · **Blocks:** M3

New phase between `gap-claim` and code, encoded in `chump --execute-gap`
and the orchestrator's dispatch path:

1. Agent enumerates files it expects to touch (LLM call, structured output:
   `{files: [...], rationale: "..."}`).
2. Agent runs `gh pr list --json number,headRefName,files --state open` and
   computes overlap with the planned file set.
3. If ≥2 open PRs touch any planned file → abort with
   `queue too crowded, try a different gap`. The dispatcher picks the next
   gap from the priority list; the original gap returns to the pool.
4. If 0–1 overlaps → write `.chump-plans/<gap-id>.md` containing planned
   files, conflict score, 5-line design summary. Plan auto-attaches to the
   PR description at `bot-merge.sh` time.

Cost: one cheap Sonnet/Haiku call per dispatch. Saves the rebase tax we're
paying today (often >30 min per cascading DIRTY event).

**Retirement (post-ship):** `gap-claim.sh` alone is no longer sufficient
for autonomous dispatch — `chump --execute-gap` must produce a plan first.
Manual local development is unchanged.

**Acceptance:**
- `chump --execute-gap` produces `.chump-plans/<gap>.md` before any code edit.
- Dispatcher aborts when ≥2 open PRs touch any planned file (verified with
  a synthetic test: open 2 dummy PRs touching `src/foo.rs`, dispatch a gap
  that lists `src/foo.rs` → abort, no commits made).
- Plan body appears verbatim in PR description after `bot-merge.sh`.

### M3 — Stacked-PR dispatcher (INFRA-061)

**Effort:** 3–4 days · **Depends on:** M1, M2

Add `bot-merge.sh --gap X --stack-on <prev-gap>` for related work.
Mechanics:

- New flag resolves `<prev-gap>` to the open PR's branch + head SHA.
- The stacked PR's base branch becomes the prev PR's head, not `main`.
- When prev lands, the merge queue auto-rebases stacked descendants.
- `chump gap depends_on` already exists in the YAML schema; add a `--stack`
  option to `chump gap reserve` that enforces `depends_on` at the DB level
  and emits `--stack-on` for the dispatcher.

Reference implementations: Graphite (`gt`), Sapling, ghstack. We don't need
their full UX — just one-deep stacks for the common dispatcher case.

**Retirement (post-ship):** mass refactors no longer fan out as
independent parallel PRs. Document the stack pattern in `AGENTS.md`.

**Acceptance:**
- `bot-merge.sh --stack-on INFRA-XYZ` opens a PR with `base=claude/<prev>`.
- Landing the prev PR triggers auto-rebase of the stacked PR (verified
  end-to-end with two synthetic gaps).

### M4 — Feature flags for cognitive-architecture work (INFRA-062)

**Effort:** ~1 week · **Parallelizable with M3**

Long-lived COG-* branches die. Add `src/runtime_flags.rs` with:

- `CHUMP_FLAGS=cog_040,cog_041` env-var parsing at startup.
- `if flags::is_enabled("cog_040")` gates around new behavior.
- A flag default policy (default-off until cycle review + bench
  confirmation, then default-on, then code-cleanup PR removes the flag).
- Bench harness runs **two** builds per benchmark: flags-off baseline vs
  flags-on candidate. COG reflection rows tag the flag set under test.

Modeled on Google/Meta trunk-based + dark-launch. Removes the rebase tax
on cognitive work entirely.

**Retirement (post-ship):** branch hygiene rule moves from "long branches
ok for COG" to "all COG work merges to main behind a flag within 1 day of
gap claim". Update `CLAUDE.md` Hard Rules.

**Acceptance:**
- `runtime_flags::is_enabled("foo")` reads `CHUMP_FLAGS=foo` correctly.
- Two ship cycles of a real COG-* gap have used the flag pattern (replacing
  what would otherwise be a 3+ day branch).
- `cargo bench` produces flag-on vs flag-off rows.

### M5 — Cycle-time dashboard + cost-routed dispatcher (INFRA-063)

**Effort:** 2–3 days · **Independent of M2-M4**

Two integrations on top of existing infra:

- **`chump dashboard`** — reads reflection rows + git log + `gh pr list`
  to print: PRs landed today/week, median cycle-time gap-claim → main,
  rebase-minutes burned, dispatcher backend split (Sonnet vs Together
  vs local), top 5 stale worktrees. One screen. Daily ritual.
- **Cost-routed dispatch** — `CHUMP_DISPATCH_BACKEND` already supports
  `claude` and `chump-local`. Add a router that picks per-gap by
  `priority + effort + domain`: trivial xs gaps → Together free tier;
  design/architecture gaps (P1, l/xl effort) → Opus; refactors → Sonnet.
  Preserves manual override.

**Acceptance:**
- `chump dashboard` produces the listed metrics from real data.
- Dispatcher logs show backend selection rationale per gap.
- One week of operation shows a measurable cost split (target: ≥40% of
  dispatches on the cheap tier without quality regression).

## Success criteria for the roadmap

After all five ship:
1. `--skip-tests` is no longer used in `bot-merge.sh` runs (corruption sources eliminated).
2. Median cycle time gap-claim → main < 90 minutes (down from current 3-6 hours).
3. ≤2 stale worktrees at any time (reaper handles steady state, planning prevents fanout).
4. Zero gaps.yaml merge conflicts in any 7-day window.
5. New COG-* gaps land within 24h of claim, behind a flag.
