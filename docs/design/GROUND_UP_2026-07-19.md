# Ground-up: what Chump looks like designed from scratch (2026-07-19)

> Written the day the operator asked "how would you do this from the ground up?"
> — which happened to be the day an active Opus session fought: the worktree-db
> phantom-ship bug (CREDIBLE-154), the launchd disabled-override graveyard
> (RESILIENT-168/174), a ghost integration-queue entry, the canonical-checkout
> branch hijack (RESILIENT-175), a 41 GB target/debug disk treadmill, and ~10
> pre-commit/CI gates on the way to shipping two P0s. Every finding below has a
> same-day receipt.

## The one-line diagnosis

Every serious bug was the same bug in a different costume: **state that lives
in many places, updated by claims instead of observations.**

- worktree-local `.chump/state.db` vs canonical → phantom ships, lost
  ready_to_ship, gap-ID collisions (INFRA-3338, fired live during this session)
- launchd override DB vs plists → daemons dead through months of "reinstalls"
- YAML mirrors vs SQLite → hijack-guard false positives, import wedges,
  pull-clobbers
- worker logs saying "shipped" vs GitHub saying nothing → false throughput
- scoreboard saying "auto-deploy done" vs a 13-day-stale installed binary
  (CREDIBLE-151)

Fix that one disease and most of the wedge taxonomy disappears.

## The design

### 1. `chumpd` — one supervising daemon owns everything mutable

A single Rust process holding:
- **state** — SQLite embedded; *nobody else opens the file*. CLI, workers,
  ChumpBar, PWA, hooks talk to a local socket API. Deletes the entire
  resolve-db-by-cwd class by construction.
- **workers** — spawned as supervised child processes. No tmux zoo, no farmer
  daemon reviving what the supervisor should own, no launchd override DB.
- **the event log** — append-only, the only bus (today's ambient.jsonl,
  promoted). A2A, votes, alerts, wake events: all just events.
- **the API** — everything the operator or an agent does is an API call and
  therefore an event with provenance.

### 2. Transitions require receipts; "done" is observed, never reported

A worker never says "shipped." It pushes a branch; `chumpd` — fed by the
GitHub webhook stream we already have (github_cache.db) — *observes* PR open,
CI pass, merge, and flips the gap itself with the merge SHA as evidence.
CREDIBLE-092 (webhook flips merged-PR gaps, shipped #3144) is the seed of this
layer. Reality-Check doctrine ("a detector firing is a SIGNAL, not an
OUTCOME") stops being a rule agents must remember and becomes a property the
system enforces.

### 3. One policy engine instead of the gate archipelago

~18 grep-based pre-commit/CI tripwires, each with its own bypass env and
trailer format, each encoding a real incident. Keep the incidents, unify the
mechanism: `chump verify` — one binary, typed rules over **parsed semantics**
(a YAML quote-style change can't be a "title hijack" when you compare values),
identical locally and in CI (INFRA-1670/1867 finish this), machine-readable
remediation per rule, bypass = a signed event in the log. Preflight parity
stops being a maintenance chore because there is only one implementation.

### 4. Kill the YAML mirrors

Already "optional post-INFRA-760"; in practice they generate false hijacks,
prereg-gate friction on untouched files, import wedges (duplicate `evidence`
field), and pull-clobber fights. SQLite canonical + a versioned dump for git
history. One representation of a gap.

### 5. The substrate is not a laptop

RESILIENT-169 (wake recovery), chump-mode, pmset rules, disk reapers — all
compensation for running a datacenter on a machine designed to sleep in a
backpack. End-state (already the roadmap's INFRA-1543 direction): `chumpd` on
an always-on box (Pi-mesh / mini / the self-hosted-runner host); the MacBook
is a *client*. Lid physics stop being fleet physics.

### 6. Outcomes first-class, gaps derived

A maintenance-shaped gap schema generates maintenance-shaped work (the
93%-meta finding). The outcome tree is the primary object; gaps are edges
toward an outcome (outcome_id mandatory — backfilled to 100% on 2026-07-19);
the picker ranks by outcome-weighted unblock count.

## What stays

The four pillars. Evidence-and-receipts culture. The doctrine docs
(REALITY_CHECK, DURABLE_FIX). Effort-scaled timeouts, effort-class model
routing, webhook cache, auto-merge discipline, ambient-stream peripheral
vision. The ideas are right; the execution substrate (files + cwd + launchd +
shell glue) betrays them.

## The migration IS the rewrite

The repo is already converging here piecewise: META-107 moved the substrate
into Rust crates, `chump-integrator` is a proto-`chumpd`, preflight-parity is
the policy engine, the Outcome FK landed, webhook→cache is the observation
layer. Sequence (each step deletes a bug class met on 2026-07-19):

| # | Step | Gap | Kills |
|---|---|---|---|
| 1 | Ship-state by observation (fix/delete Mode A enqueue; webhook flips gaps; worker "shipped" requires PR evidence) | **CREDIBLE-154** (P0) | phantom ships, re-pick waste, false throughput |
| 2 | `chumpd` supervisor umbrella — fold worker spawn + daemon zoo into one supervised process tree | **MISSION-051** | disabled-override graveyard, farmer, 43-plist sprawl |
| 3 | Retire YAML gap mirrors (SQLite + versioned dump) | **ZERO-WASTE-020** | hijack false-positives, import wedges, pull-clobbers |
| 4 | `chump verify` — unify gates, parsed-semantics rules, single impl local+CI | **CREDIBLE-155** | gate archipelago, parity drift, bypass sprawl |
| 5 | Substrate off-laptop: `chumpd` on the always-on runner host | **RESILIENT-176** (extends INFRA-1543) | lid physics, disk treadmill, sleep outages |

Operator decision 2026-07-19: Opus builds directly where it speeds things up;
the fleet picks up the rest. Feed-the-fleet defaults suspended for this
migration where speed wins.
