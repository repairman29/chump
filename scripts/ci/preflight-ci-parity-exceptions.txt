# Explicit allowlist for CI gates that are NOT mirrored in `chump preflight`
# and are not classified Tier-D in docs/process/CI_GATES_INVENTORY.md.
#
# These are Tier-C gates (mirrorable, follow-up gap filed) and infrastructure
# setup steps (Tier-D in spirit but not yet documented there). Adding a gate
# here declares "we know this is not mirrored; a follow-up gap exists or this
# is intentionally deferred."
#
# REMOVING an entry from this file without adding a mirror in src/preflight.rs
# will cause test-preflight-ci-parity.sh to fail — that is intentional.
# ADDING a new entry here for a genuinely mirrorable gate must come with a
# filed follow-up gap ID in the comment.
#
# Format:
#   <script-path-or-step-name-fragment>  # reason: <one-line why> [gap: <ID>]
#
# Matching rules (in test-preflight-ci-parity.sh):
#   - If a gate's run: command contains the entry → allowlisted
#   - If a gate's step name contains the entry → allowlisted
#   - Exact match preferred; substrings accepted for compound step names
#
