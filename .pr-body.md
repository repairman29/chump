## Summary

Ground-up step 3 (`docs/design/GROUND_UP_2026-07-19.md`): retires the per-file
YAML gap mirrors (`docs/gaps/<ID>.yaml`, including the `docs/gaps/closed/`
archive — ~1625 files total). `.chump/state.db` is canonical; the
already-existing `.chump/state.sql` tracked dump (with its own merge driver,
INFRA-310) is the git-history surface. `docs/gaps/README.md` is a tombstone
documenting the new model and how to recover a gap's pre-retirement history.

Per-file mirrors were "optional post-INFRA-760" but stayed a live write path
via `chump gap reserve/set/ship --update-yaml` and `decompose`, generating:
- false hijack blocks on cosmetic YAML quote-style churn (CREDIBLE-153)
- import wedges from duplicate `evidence` fields (20 files fixed in PR #3206)
- prereg-gate friction on untouched files
- pull-clobber races between two representations of the same gap

An earlier attempt at this exact retirement (INFRA-496/498/499, referenced in
several code comments this PR touches) got partway there — it disabled the
drift-detector logic defensively but never actually deleted the files, so
`chump gap reserve`/`set`/`ship --update-yaml`/`decompose` kept recreating
them. This PR finishes it: the write call sites are gone, not just
directory-existence-gated.

### Found in passing

Two of the CI tests I touched used a live `chump-binary | grep -q pattern`
pipe. Under `set -o pipefail`, `grep -q` exits the instant it finds a match —
if the producer is still writing, it gets SIGPIPE, and pipefail reports the
pipeline as failed even though the match *was* found. Reproduced ~1-in-5 runs
locally. Fixed by capturing output to a variable before grepping (the
herestring/variable-capture pattern CLAUDE.md's pipefail-sweep rule already
asks for).
